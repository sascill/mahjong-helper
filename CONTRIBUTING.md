# Contributing

## 작업 흐름

모든 기능은 다음 순서로 진행합니다.

```text
스펙 작성
→ 테스트 코드 작성
→ 실제 구현
→ 리뷰
```

- 스펙을 먼저 작성하고 구현 범위와 기대 결과를 확정합니다.
- 스펙을 검증할 수 있는 테스트 코드를 실제 구현보다 먼저 작성합니다.
- 테스트를 기준으로 기능을 구현합니다.
- 구현이 끝나면 코드와 결과가 스펙을 충족하는지 리뷰합니다.

## 커밋 규칙

- `core:` 프로젝트 설정 및 핵심 기반 변경
- `feat:` 새로운 기능 추가
- `fix:` 오류 수정
- `docs:` 문서 추가 및 수정
- `refactor:` 기능 변경 없는 코드 구조 개선
