# 손패 선택 기능 스펙

## 목적

사용자가 첫 손패 13장과 장풍·자풍을 입력할 수 있게 한다.

이 기능은 입력값을 생성하는 역할만 담당하며 역 추천이나 화면 이동 규칙은 직접 구현하지 않는다.

## 입력과 출력

```ts
type HandSelectionProps = {
  onAnalyze: (input: HandInput) => void;
};
```

유효한 입력 상태에서 사용자가 분석 버튼을 누르면 `HandInput`을 `onAnalyze`로 전달한다.

## 화면 구성

화면 요소는 다음 순서로 배치한다.

1. 현재 선택한 손패와 선택 장수
2. 장풍과 자풍 선택
3. 만수·통수·삭수·자패 선택 영역
4. 초기화 및 분석 버튼

시각 디자인은 별도로 정의하지 않는다. 모바일에서 각 패와 버튼을 손가락으로 선택할 수 있는 크기를 확보한다.

## 초기 상태

- 선택한 패는 0장이다.
- 장풍은 선택되지 않은 상태다.
- 자풍은 선택되지 않은 상태다.
- 분석 버튼은 비활성화한다.

## 패 선택

- 34종 패를 무늬별로 구분하여 표시한다.
- 패 버튼을 누르면 해당 패 한 장을 손패에 추가한다.
- 선택한 손패는 도메인의 정렬 규칙에 따라 표시한다.
- 같은 패는 최대 4장까지 선택할 수 있다.
- 전체 손패는 최대 13장까지 선택할 수 있다.
- 더 이상 추가할 수 없는 패 버튼은 비활성화한다.
- 현재 선택한 장수를 `현재 장수 / 13` 형식으로 표시한다.

## 패 제거

선택한 손패에서 패 하나를 누르면 해당 패 한 장을 제거한다.

같은 패가 여러 장 있더라도 한 번의 동작으로 한 장만 제거한다.

## 장풍과 자풍

장풍과 자풍은 각각 다음 값 중 하나를 선택한다.

```text
동·남·서·북
```

- 장풍과 자풍은 독립적으로 선택한다.
- 같은 풍을 동시에 선택할 수 있다.
- 기본값을 임의로 지정하지 않는다.

## 초기화

초기화 버튼을 누르면 다음 값을 모두 초기 상태로 되돌린다.

- 선택한 패
- 장풍
- 자풍

## 분석 버튼

다음 조건을 모두 만족할 때만 분석 버튼을 활성화한다.

- 패가 정확히 13장이다.
- 장풍이 선택되어 있다.
- 자풍이 선택되어 있다.
- 손패 도메인 검증을 통과한다.

활성화된 분석 버튼을 누르면 현재 값을 `HandInput`으로 만들어 `onAnalyze`에 전달한다.

추천 계산과 추천 결과 화면 이동은 상위 페이지에서 처리한다.

## 유니코드 표시

패의 모양은 손패 도메인에 정의된 `symbol`을 사용한다.

선택 동작은 유니코드 문자 자체가 아닌 `TileId`를 기준으로 처리한다.

## 테스트 기준

- 초기 상태에 선택한 패가 없다.
- 패 버튼을 누르면 손패에 한 장이 추가된다.
- 선택한 손패의 패를 누르면 한 장이 제거된다.
- 같은 패를 5장 이상 선택할 수 없다.
- 전체 손패를 14장 이상 선택할 수 없다.
- 선택한 손패가 정해진 순서로 표시된다.
- 장풍과 자풍을 독립적으로 선택할 수 있다.
- 초기화하면 패와 장풍·자풍이 모두 초기화된다.
- 입력이 완성되기 전에는 분석 버튼이 비활성화된다.
- 13장과 장풍·자풍이 모두 입력되면 분석 버튼이 활성화된다.
- 분석 버튼을 누르면 현재 `HandInput`이 전달된다.
