# 디자인 스펙

이 문서는 애플리케이션 전체에 공통으로 적용하는 시각 언어와 UI 원칙을 정의한다.

Stitch 시안은 디자인 방향을 정하는 참고 자료로만 사용한다. 생성된 코드, 외부 글꼴, 이미지와 아이콘은 프로젝트에 직접 복사하지 않는다.

## 디자인 목표

- 초보자가 첫 화면에서 해야 할 행동을 바로 이해할 수 있어야 한다.
- 마작의 분위기는 살리되 게임 화면처럼 복잡하거나 화려하게 만들지 않는다.
- 패와 조건처럼 중요한 정보가 장식보다 먼저 보여야 한다.
- 실제로 동작하지 않는 버튼이나 메뉴를 미리 노출하지 않는다.

## 시각 방향

짙은 녹색과 따뜻한 흰색을 기본으로 사용하고, 중요한 행동에는 붉은색을 제한적으로 사용한다.

마작패와 주요 카드는 작은 모서리와 아래쪽 그림자를 사용해 손으로 만지는 물체처럼 표현한다. 일반 정보 영역은 그림자를 최소화해 패가 가장 눈에 띄게 한다.

## 색상 토큰

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--color-background` | `#f7f7f4` | 앱 배경 |
| `--color-surface` | `#ffffff` | 카드와 패 |
| `--color-surface-muted` | `#eceeea` | 보조 영역 |
| `--color-primary` | `#073b2a` | 브랜드와 주요 강조 |
| `--color-primary-strong` | `#002d1c` | 헤더와 강한 배경 |
| `--color-primary-soft` | `#c7ead5` | 선택 상태와 보조 강조 |
| `--color-accent` | `#b71032` | 분석 등 핵심 행동 |
| `--color-accent-pressed` | `#8f0826` | 핵심 행동의 그림자와 눌림 |
| `--color-gold` | `#c9a227` | 제한적인 마작 분위기 강조 |
| `--color-text` | `#191c1a` | 기본 텍스트 |
| `--color-text-muted` | `#626b65` | 설명과 보조 텍스트 |
| `--color-border` | `#cfd5d0` | 구분선과 입력 테두리 |

색상은 의미에 맞는 토큰으로 사용한다. 컴포넌트에 같은 색상 값을 반복해서 직접 작성하지 않는다.

## 글꼴

- 기본 글꼴은 `Pretendard`와 운영체제의 산세리프 글꼴을 사용한다.
- 외부 글꼴을 실행 시점에 불러오지 않는다.
- 제목은 굵고 짧게 작성하며 본문은 한 번에 읽을 수 있는 길이로 유지한다.
- 패 개수와 판수처럼 숫자 정렬이 중요한 정보는 고정폭 글꼴을 사용할 수 있다.

## 레이아웃

- 모바일 단일 열을 기준으로 한다.
- 콘텐츠 최대 너비는 `600px`로 제한한다.
- 화면 좌우 여백은 최소 `16px`로 둔다.
- 간격은 `4px` 단위를 기준으로 `4`, `8`, `12`, `16`, `24`, `32px`를 사용한다.
- 누를 수 있는 요소는 최소 `44px` 높이와 너비를 확보한다.
- 하단 주요 메뉴가 있는 화면은 메뉴에 콘텐츠가 가리지 않도록 하단 여백을 확보한다.

## 컴포넌트 원칙

### 마작패

- 유니코드 문자를 흰색 패 컨테이너 안에 표시한다.
- 패는 작은 모서리와 아래쪽 그림자로 입체감을 준다.
- 선택 가능, 선택됨, 비활성 상태를 색상과 명도로 구분한다.
- 문자 자체와 패 표시 컴포넌트를 분리해 향후 이미지 에셋으로 교체할 수 있게 한다.

### 버튼

- 화면의 핵심 행동은 한 개만 강하게 강조한다.
- 주요 분석 행동은 붉은색, 일반 이동과 브랜드 행동은 녹색을 사용한다.
- 비활성 버튼은 동작할 수 없다는 점이 명확해야 한다.
- 장식용 버튼이나 아직 구현되지 않은 이동 버튼은 만들지 않는다.

### 카드와 입력

- 관련된 정보만 하나의 카드로 묶는다.
- 입력 요소는 흰색 표면과 분명한 테두리를 사용한다.
- 포커스 상태를 키보드와 터치 환경 모두에서 식별할 수 있게 한다.

## 내비게이션

공통 앱 셸은 브랜드 헤더와 하단 주요 메뉴를 제공한다.

현재 구현된 경로만 메뉴에 노출한다. 역 도감 경로가 구현되면 같은 내비게이션에 추가한다.

## 제품 범위와의 관계

디자인은 다음 제품 원칙을 바꾸지 않는다.

- 첫 손패는 13장과 장풍·자풍을 입력한다.
- 결과는 목표로 삼을 역을 추천하며 버릴 패를 지시하지 않는다.
- 실제 화료 확률, 부수, 점수와 도라는 표시하지 않는다.
- 사진 인식과 실제 패 이미지 에셋은 현재 범위에 포함하지 않는다.
